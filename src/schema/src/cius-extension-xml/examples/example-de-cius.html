Introduction

This CIUS-DE-NAT builds on top of the European Norm EN 16931-1:2017. All rules of the underlying specification apply with the addition of the rules stated in this document.

Indetifying kosit@finanzen.bremen.de

An identification of the specification containing the total set of rules regarding semantic content, cardinalities and business rules to which the data contained in the instance document conforms is reported in BT-24 “Specification identification”. The identifier for this CIUS is:

BT-24 1..1 Specification identification urn:ce.eu:en16931:2017:xoev-de:kosit:standard:xrechnung_1.1

According to the e-Invoice instance document syntax the specification identification will be:

New rules imposed by this specification

[BR-DE-1] Eine Rechnung (INVOICE) muss Angaben zu „PAYMENT INSTRUCTIONS“ (BG-16) enthalten. [BG-16]
[BR-DE-1] The INVOICE shall contain information on PAYMENT INSTRUCTIONS (BG-16). [BG-16]

[BR-DE-4] The element “Seller post code” (BT-38) shall be transmitted. [BT-38]

[BR-DE-6] The element „Seller contact telephone number“(BT-BT) shall be transmitted. [BT-42]

[BR-DE-13] In der Rechnung müssen Angaben zu einer der drei Gruppen „CREDIT TRANSFER“ (BG-17), „PAYMENT CARD INFORMATION“ (BG-18) oder „DIRECT DEBIT“ (BG-19) übermittelt werden. [BG-17, BG-18, BG-19]
[BR-DE-13] The invoice shall contain information on one of the three groups: “CREDIT TRANSFER” (BG-17), “PAYMENT CARD INFORMATION” (BG-18) or “DIRECT DEBIT” (BG-19). [BG-17, BG-18, BG-19]
[BR-DE-14] Das Element „VAT category rate“ (BT-119) muss übermittelt werden. [BT-119]
[BR-DE-14] The element “VAT category rate” (BT-119) shall be transmitted. [BT-119]
[BR-DE-15] Das Element „Buyer reference“ (BT-10) muss übermittelt werden. [BT-10]
[BR-DE-15] The element „Buyer reference“ (BT-10) shall be transmitted. [BT-10]
[BR-DE-16] Das Element „Seller VAT identifier“ (BT-31) muss übermittelt werden, wenn nicht das Element „Seller tax registration identifier“ (BT-32) oder eine Gruppe „SELLER TAX REPRESENTATIVE PARTY“ (BG-11) übermittelt wurden. [BT-31, BT-32, BG-11]
[BR-DE-16] The element “Seller VAT identifier” (BT-31) shall be transmitted, if the element “Seller tax registration identifier” (BT-32) or a group „SELLER TAX REPRESENTATIVE PARTY“ (BG-11) is not transmitted. [BT-31, BT-32, BG-11]
[BR-DE-17] Mit dem Element „Invoice type code“ (BT-3) sollen ausschließlich folgende Codes aus der Codeliste UNTDID 1001 übermittelt werden: 326 (Partial invoice), 380 (Commercial invoice), 384 (Corrected invoice), 381 (Credit note) [BT-3]
[BR-DE-17] With the element “Invoice type code” (BT-3) only the following codes from the code list UNTDID 1001 shall be transmitted: 326 (Partial invoice), 380 (Commercial invoice), 384(Corrected invoice), 381 (Credit note) [BT-3]
[BR-DE-18] Informationen zur Gewährung von Skonto oder zur Berechnung von Verzugszinsen müssen wie folgt im Element „Payment terms“ (BT-20) jeweils in einer eigenen Zeile übermittelt werden: Anzugeben ist im ersten Segment „SKONTO“ oder „VERZUG“, im zweiten „TAGE=n“, im dritten „PROZENT=n“, wobei die Segmente jeweils von einer „#“ umfasst sind. Prozentzahlen sind mit Punkt getrennt von zwei Nachkommastellen anzugeben. Liegt dem zu berechnenden Betrag nicht BT-115, „fälliger Betrag“ zugrunde, sondern nur ein Teil des fälligen Betrags der Rechnung, ist der Grundwert zur Berechnung von Skonto oder Verzugszins als viertes Segment „BASISBETRAG=n“ mit dem semantischen Datentyp Amount anzugeben. [BT-20, BT-115]
[BR-DE-18] Information on any discount or interest on late payments shall be given in the element “Payment terms” (BT-20) in a separate line as follows: In the first segment, state “DISCOUNT” or “LATE PAYMENTS”, in the second segment state “DAYS=n”, in the third segment state “PERCENT=n”; each segment must be enclosed in hashes (#). Percentages shall be given as a decimal number with two places after the point. If the calculated amount is not based on BT-115 “amount payable”, but is only part of the payable amount given in the invoice, the basic amount shall be stated in a fourth segment “BASIC AMOUNT=n” with the semantic data type AMOUNT for the calculation of discount or interest on late payment. [BT-20, BT-115]
[BR-DE-19] The size of all attached documents together may not exceed 15MB []
